WEALTH PACK - Signal Type Vocabulary

Valid signal types for wealth pack:

1. risk_tolerance_change
   - Client's risk tolerance has changed
   - Payload: { client_id: string, previous_tolerance: string, new_tolerance: string, reason: string }
   - Example: "Client moved from Aggressive to Moderate following retirement"

2. large_transaction_alert
   - Significant transaction that may require review
   - Payload: { client_id: string, transaction_type: string, amount: number, description: string }
   - Example: "Client withdrew $500,000 for real estate purchase"

3. beneficiary_update
   - Change to beneficiary designations
   - Payload: { client_id: string, account: string, change_type: string, details: string }
   - Example: "Added spouse as primary beneficiary on IRA"

4. tax_event
   - Significant tax event requiring action
   - Payload: { client_id: string, event_type: string, estimated_impact: number, deadline: string }
   - Example: "Capital gains distribution of $75,000 expected in December"

5. estate_document_update
   - Estate planning document has been updated
   - Payload: { client_id: string, document_type: string, update_date: string, summary: string }
   - Example: "Trust agreement amended to add grandchildren"

6. investment_objective_change
   - Client investment objectives have changed
   - Payload: { client_id: string, previous_objective: string, new_objective: string, reason: string }
   - Example: "Shifted from growth to income focus"

7. account_ownership_change
   - Change in account ownership or structure
   - Payload: { client_id: string, account: string, previous_ownership: string, new_ownership: string }
   - Example: "Converted individual account to joint with spouse"

8. compliance_violation_flag
   - Potential compliance issue identified
   - Payload: { client_id: string, violation_type: string, details: string, severity: string }
   - Example: "Pattern trading detected in client account"
